# Prompt: Extensibility / Plugin Model

Specify a plugin/extension framework for policy, action, and adapter modules.

Inputs:
- plugin manifest (name, version, capabilities, dependencies)
- core platform interface contract

Outputs:
- plugin lifecycle design (install/enable/disable/update)
- compatibility checks
- security isolation rules

Requirements:
- include plugin discovery and certification process
- include explicit API versioning guarantees
- include rollback and quarantine strategy
